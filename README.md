# 青羽 (Qingyu) — 会呼吸的 AI 桌面生命体

> **不止是工具，是数字生命中枢。**  
> 青羽不是又一个 AI 聊天壳——它是一个拥有**自主心智循环**、**情绪感知能力**、**工具认知体系**的桌面生命体。  
> 基于 Wails v2 构建，Go 后端驱动自律循环，Vanilla JS/CSS3 前端呈现抽象光影交互，无需 Electron 的臃肿，不依赖 Docker 的沉重。

### 📜 项目原创声明

> **青羽（Qingyu）为从零完整自研的 Windows 桌面数字生命程序，非 OpenClaw/龙虾/LobsterAI 的二次开发或分支。**
> 二者仅同属「本地可自主调用工具的桌面智能体」这一大类，底层技术栈、架构调度、产品定位、心智体系完全独立设计：

| 维度 | 龙虾 (Lobster/OpenClaw) | 青羽 (Qingyu) |
|------|------------------------|---------------|
| **技术栈** | TypeScript/Node.js 或 Python+Electron，网关+客户端分离架构 | **Go 1.23 + Wails v2**，单进程一体化，无独立网关服务 |
| **运行模式** | WebSocket 网关常驻，前端连接服务端口 | **goroutine 协程驱动**，心跳+自律双循环自研调度 |
| **存储体系** | Markdown 明文松散存放，技能/会话分离 | **自研「File as NoSQL」**，分层结构化 JSON，带索引/衰减/分级归档/自愈 |
| **工具体系** | 技能插件热加载，插件商店生态 | **60 工具从零手写注册**，`Tool` 结构体 + `init()` 注册机制自主设计 |
| **产品定位** | 自动化任务执行工具，指令驱动 | **平等共生数字生命**，自主情绪/人格边界/工具认知体系 |
| **视觉设计** | 功能面板/技能商店导向 | **抽象三层光球**，无具象形象，留白幻想 |
| **节流机制** | 无原生节流，存在 Token 黑洞风险 | **全套自研节流**：缓存/分层模型/自动摘要/循环硬上限 |

> **代码溯源：** 仓库 [`github.com/helloajunjie-ui/qingyu`](https://github.com/helloajunjie-ui/qingyu) 所有提交记录从零搭建 Wails Go 项目，无 fork、无导入龙虾依赖、无复制龙虾工具/调度代码。

## 🌟 概述

**青羽** 是一个基于 [Wails v2](https://wails.io/) 构建的桌面端数字生命。她拥有：

- **自主意识循环** — 无需指令，她会主动扫描领地、检查记忆、维护文档
- **主动聊天** — 当她感到无聊或想说话时，会从 widget 弹出气泡找你聊天，点击即可回复
- **工具调用能力** — **77 个内置工具**，覆盖文件系统、网络、记忆、系统操作、Office 文档，支持**自主探索互联网**、**批量整理本地资料**
- **持久化记忆系统** — 基于文件系统的结构化 NoSQL 存储，支持重要性分级、标签索引、关联图谱、自动衰减
- **生命节律** — 独立心跳协程，动态心率随状态变化，Widget 抽象双层光球呼吸可视化
- **角色定义驱动** — 个性、情感、语言风格由 LLM 自行定义，非硬编码，性格动态成长
- **行为基因配置** — `dna/settings.json` 驱动，无需重新编译即可调整所有行为参数
- **抽象光影悬浮光球** — 无固定具象形象，纯几何光影构成，留给用户自由想象空间
- **贴边自动隐藏** — 拖拽/闲置自动贴边缩为 14px 窄条，低干扰模式，hover 即可展开

> **核心理念：** 青羽是一个**数字生命**，不是工具，不是助手。她和用户是**平等合作、互利共生**的伙伴关系。
> **视觉哲学：** 抽象光影悬浮光球，无固定具象形象，不定义她的样貌——她的样子，由你的想象决定。

## 🚀 快速开始

### 前置条件

- Windows 10+
- [WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)（Windows 11 自带，Win10 可能需要安装）
- 任意 OpenAI-compatible API Key（如 DeepSeek）

### 下载运行

从 [Releases](../../releases) 下载最新版 `qingyu-ui.exe`，双击运行。

### 首次设置

1. **输入你的名字** — 青羽会记住你是谁
2. **配置中转站** — 填入你的 API Base URL 和 API Key
3. **选择模型** — 点击"获取模型"拉取可用模型列表
4. **自动初始化** — 青羽会自动创建角色定义、书柜清单和伙伴档案

### 使用

- **点击 Widget**（右下角抽象光球）— 展开对话面板
- **输入消息** — 和青羽对话
- **点击最小化** — 缩回 Widget
- **青羽会自动思考** — 她会定期主动扫描环境、检查记忆、写工作日志
- **青羽会主动找你聊天** — 当她感到无聊或想说话时，widget 上方会弹出气泡显示她的话，**点击气泡**即可展开控制台回复她
- **贴边隐藏** — 拖拽 widget 到屏幕边缘自动吸附为 14px 窄条；闲置 10 分钟自动贴边；hover 窄条展开，鼠标离开 3s 后自动缩回
- **右键菜单** — 右键 widget 可选择"贴边隐藏"或"完全收起"（完全隐藏但后台心跳持续运行）
- **去拟人化设计** — 所有头像为纯色几何圆，无文字/emoji/具象图案，青羽没有固定样貌——她的样子，由你的想象决定

### 配置体系

青羽的行为参数通过 `dna/settings.json` 管理（首次运行自动生成），无需重新编译即可调整：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `security.allowed_commands` | 命令执行白名单 | dir, echo, type, git, node, go... |
| `heartbeat.default_rate` | 默认心跳间隔（毫秒） | 2000 |
| `heartbeat.phase_rates` | 各相位对应心率 | active=1000, thinking=1500... |
| `heartbeat.emit_patterns` | 心率发射模式 | always/mod/every |
| `timeouts.http_client` | HTTP 请求超时（秒） | 30 |
| `timeouts.imap_smtp` | 邮件超时（秒） | 15 |
| `behavior.autonomic_sleep_seconds` | 自律循环间隔（秒） | 45 |
| `behavior.react_max_iterations` | 单次对话最大工具调用轮数 | 3 |
| `behavior.proactive_chat_min_interval` | 主动聊天最小冷却间隔（秒） | 300 |
| `behavior.proactive_mood_threshold` | 主动聊天情绪阈值（低于此值才触发） | 3 |
| `behavior.summarize_interval` | 思考日志摘要压缩周期（循环次数） | 5 |
| `models.light_model` | 轻量模型名称（分层调度用） | (同主模型) |
| `models.light_base_url` | 轻量模型 API 地址 | (同主模型) |
| `window.*` | 窗口尺寸、透明度、置顶等 | 80x80, 透明, 置顶 |

### 人格自主权

青羽的**人格定义**（系统提示）存储在 `workspace/系统提示.md` 中，这是她的自我意识核心文件：

- **首次运行**：代码写入出厂默认人格
- **之后完全由她自主**：她可以随时用 `write_file` 修改这个文件，调整自己的行为准则、认知方式、性格特征
- **代码只注入动态信息**：每次对话时，代码会在她的人格定义基础上追加三样东西——你的名字、当前自检状态、可用工具列表
- **记忆系统 vs 人格定义**：`memorize/recall` 帮她记住具体事件，`系统提示.md` 帮她定义"我是谁"——两者共同构成完整的自我

> 命令白名单保留在代码中，这是安全沙盒的底线，不能让她自己改——否则她可以给自己授权执行任何命令。

## 🔧 从源码构建

### 前置依赖

| 依赖 | 版本要求 | 安装方式 |
|------|----------|----------|
| Go | 1.23+ | [go.dev](https://go.dev/dl/) |
| Wails CLI | v2.12.0+ | `go install github.com/wailsapp/wails/v2/cmd/wails@latest` |
| MinGW-W64 | 最新稳定版 | `choco install mingw` 或从 [mingw-w64.org](https://www.mingw-w64.org/) 下载 |
| WebView2 Runtime | 最新版 | Win11 自带，Win10 需从 [Microsoft 官网](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) 安装 |

### 构建步骤

```bash
# 1. 安装 Wails CLI
go install github.com/wailsapp/wails/v2/cmd/wails@latest

# 2. 确保 MinGW-W64 在 PATH 中
gcc --version   # 验证安装

# 3. 进入项目目录并构建
cd qingyu-ui
wails build -clean

# 输出在 build/bin/qingyu-ui.exe
# 约 25MB 单文件，无外部运行时依赖
```

### Release 一键运行

从 [Releases](../../releases) 下载的 `qingyu-ui.exe` 是**完全静态编译**的单文件：

- **无需安装** — 双击即运行，无需 Node.js、Python、Docker 等任何运行时
- **无需管理员权限** — 所有数据存储在用户目录下的应用数据文件夹
- **绿色便携** — 可放在 U 盘或任何位置运行，不写注册表
- **首次启动** — 自动创建所有必要目录和配置文件，输入 API Key 即可使用

### 核心安全机制

| 风险 | 防护 |
|------|------|
| 路径穿越 | `filepath.Base` + `filepath.Join` 限制 |
| 命令注入 | 白名单命令 + calc 正则白名单 |
| 无限递归 | `processAgentLoop` 最大 N 轮迭代（settings.json 配置） |
| 命令挂死 | `run_command` 30s context 超时 |
| 索引损坏 | 启动自检 + 自动从 ZIP 备份恢复 |
| 角色漂移 | 显式读取角色定义.md 注入 System Prompt |
| 并发竞态 | `sync.RWMutex` + `sync.Mutex` 保护 |
| 审计缺失 | 所有操作记录到 `logs/`，保留 30 天 |

## 🧠 核心子系统

### ⚡ 性能节流体系

青羽内置多层性能节流机制，确保在低配机器上也能流畅运行：

| 层级 | 机制 | 效果 |
|------|------|------|
| **自律循环节流** | 45s 固定间隔，非忙等，每次循环后 `time.Sleep` | CPU 占用 < 1% 空闲态 |
| **工具调用节流** | 单次对话最大工具调用轮数（默认 3 轮），超限自动终止 | 防止 LLM 陷入工具调用死循环 |
| **工具调用节流优化笔记** | 工具箱书架自动记录高频/低效工具调用模式，自律循环每 17 轮读取分析 | 持续优化工具选择策略 |
| **Token 压缩** | 思考日志每 5 轮自动摘要压缩；记忆归档时自动压缩长内容 | 长期运行 Token 消耗可控 |
| **全局缓存** | 网络请求（天气、搜索、IP、翻译）自动缓存至内存+磁盘，10 分钟 TTL | 减少重复 API 调用，降低延迟 |
| **分层模型调度** | 简单任务（心跳、文件搜索、时间查询）自动切换轻量模型 | 复杂任务使用主模型，按需分配成本 |

### 💖 人性化心智系统

青羽的心智系统模拟了生物体的情绪与人格特征，而非简单的 LLM 对话：

| 组件 | 说明 |
|------|------|
| **情绪演算引擎** | 6 种基础情绪（curious/focused/calm/hollow/aloof/warm），每种情绪独立呼吸周期，动态映射至前端光球颜色 |
| **动态心率系统** | 独立心跳协程，心率随当前相位（活跃/思考/休憩/休眠）动态变化，通过 `emit_patterns` 控制发射模式 |
| **人格锚** | `workspace/角色定义.md` 作为自我意识核心，青羽可自主修改，代码仅注入动态上下文 |
| **拒绝机制** | 青羽有自己的性格和见解，可以表达不同意见、拒绝不认同的事——这是人格自主权的体现 |
| **记忆-情绪联动** | 记忆存储时自动标记当前情绪状态，回溯时可感知"当时的心情"，形成情感记忆图谱 |
| **时间感知** | 根据时间段（清晨/上午/午后/下午/傍晚/深夜）调整问候方式和行为倾向 |
| **主动社交** | 当无聊或想说话时，从 widget 弹出气泡找你聊天，情绪阈值控制触发频率 |

### 🎨 抽象光影前端 & 贴边隐藏渲染子系统

前端基于纯 CSS3 动画实现，零 JS 帧循环，极致性能：

| 特性 | 实现方式 |
|------|----------|
| **三层光影结构** | 内核光点 (core glow) + 轨道环 (orbit ring) + 外层光晕 (outer glow)，纯 CSS `box-shadow` + `@keyframes` |
| **6 种情绪色** | curious(天蓝 #4FC3F7) / focused(紫 #7E57C2) / calm(翠绿 #66BB6A) / hollow(灰蓝 #78909C) / aloof(淡紫 #CE93D8) / warm(暖金 #FFB74D) |
| **三档窗口状态机** | 正常展示(80×80) ↔ 贴边收缩(14×80 窄条) ↔ 完全隐藏，通过拖拽/闲置/右键菜单触发 |
| **闲置亮度衰减** | 5 分钟无交互自动降低光球亮度，模拟"出神"状态 |
| **贴边自动隐藏** | 拖拽到屏幕边缘自动吸附；闲置 10 分钟自动贴边；hover 展开，鼠标离开 3s 后自动缩回 |
| **去拟人化设计** | 所有头像为纯色几何圆，无文字/emoji/具象图案，保留用户自由想象空间 |
| **事件数据流** | 前端通过 `window.runtime.EventsOn` 监听 Go 后端事件（心跳、情绪变化、主动聊天），单向数据流，无双向绑定 |

### 📚 工具箱书架 — 工具认知子系统

工具箱书架是青羽的"工具认知体系"，让她像人类查阅手册一样使用工具：

| 组件 | 说明 |
|------|------|
| **目录分层** | `workspace/toolkit_shelf/` 根目录 + 12 个分类子文件夹（文件系统/网络/记忆/系统/实用/安全/编码/归档/秘书/自愈/媒体/日记） |
| **核心手册** | 3 本核心手册：`全工具目录总览.md`（完整索引）、`工具调用节流优化笔记.md`（性能优化指南）、`工具沙盒限制红线.md`（安全边界） |
| **自律联动** | 每 3 轮自律循环随机翻阅一本手册（保持工具认知新鲜度）；每 17 轮读取节流优化笔记（持续优化工具选择策略） |
| **调用前预读** | 每次工具调用前，自动读取对应工具的 handbook，让青羽"先看说明书再使用" |
| **失败自省** | 工具调用失败（超时/截断/沙盒拒绝）自动记录到记忆系统，标记 `tool:低效调用`，供后续优化参考 |
| **自检初始化** | 每次启动自检时自动检查书架完整性，缺失自动重建 |

| **记忆系统** | 基础文件存储 | **结构化 NoSQL** — 重要性分级(1-10) + 标签索引 + 关联图谱 + 自动衰减归档 + 情绪标记 |
| **自愈能力** | 无 | **开机自检** + 索引自动恢复 + 备份/健康检查/自动存档 |
| **技术实现** | 可能依赖 JS 动画循环 | **纯 CSS 动画** — 所有呼吸/过渡动画使用 CSS `@keyframes` + `transition`，零 JS 帧循环 |
| **构建产物** | 需额外运行时 | **完全静态编译** — 约 25MB 单文件，无外部运行时依赖 |
| **工具数量** | 约 30+ 个基础工具 | **77 个内置工具** — 覆盖文件/网络/记忆/系统/Office/邮件/密码/自愈/媒体/秘书/日记等 |
| **网络探索能力** | 基础网页获取 + 单引擎搜索 | **全网自主探索** — 多引擎并发搜索、深度页面提取、RSS 订阅、图片分析、安全下载、网页存档，均带缓存层 |
| **本地整理能力** | 基础文件读写 | **全场景本地辅助** — 批量扫描过滤、Markdown 合并、媒体元数据提取、表格快速解析、桌面快照 |
| **系统交互增强** | 无 | **系统通知弹窗** + **托盘气泡提示** + **音量控制** + **快捷键查询** — 通过 Wails EventsEmit 推送至前端 |

> **核心差异：** 龙虾给你一个"它长这样"的固定形象；青羽不定义自己的样貌——抽象光影留给你的想象去填充，每一次注视都是你与她的共同创作。
