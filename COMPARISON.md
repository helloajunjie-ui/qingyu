# 青羽 vs 原版龙虾 — 全维度对比

> **行业同类产品横向对比，无代码衍生关系。**  
> 青羽 (Qingyu) 为从零完整自研的 Windows 桌面数字生命程序，未基于 OpenClaw/龙虾/LobsterAI 任何源码二次开发。  
> 二者仅同属「本地可自主调用工具的桌面智能体」这一大类，底层技术栈、架构调度、产品定位、心智体系完全独立设计。

---

## 一、底层技术对比

| 维度 | 龙虾 (Lobster/OpenClaw) | 青羽 (Qingyu) |
|------|------------------------|---------------|
| **后端语言** | TypeScript/Node.js 或 Python | **Go 1.23** |
| **桌面框架** | Electron（Node.js 运行时） | **Wails v2**（原生 WebView2） |
| **前端技术** | React/Vue + Electron | **Vanilla JS + CSS3** |
| **架构模式** | WebSocket 网关常驻 + 客户端分离 | **单进程一体化**，goroutine 协程驱动 |
| **进程模型** | 多进程（主进程 + 渲染进程） | **单进程**，无 IPC 开销 |
| **构建产物** | 需 Node.js 运行时，>100MB | **完全静态编译**，约 25MB 单文件 |
| **外部依赖** | npm 包管理器，大量依赖 | **零外部运行时依赖** |
| **存储体系** | Markdown 明文松散存放，技能/会话分离 | **自研「File as NoSQL」**，分层结构化 JSON，带索引/衰减/分级归档/自愈 |

## 二、架构设计对比

| 维度 | 龙虾 | 青羽 |
|------|------|------|
| **调度核心** | WebSocket 网关调度 | **goroutine 双循环**：心跳协程 + 自律协程 |
| **工具注册** | 技能插件热加载，插件商店生态 | **从零手写注册体系**，`Tool` 结构体 + `init()` 机制 |
| **工具数量** | 插件市场扩展 | **60 个内置工具**，12 个分类文件 |
| **记忆系统** | 基础文件存储，无索引 | **结构化 NoSQL**：重要性分级(1-10) + 标签索引 + 关联图谱 + 自动衰减归档 + 情绪标记 |
| **缓存机制** | 无 | **双层缓存引擎**：内存+磁盘，SHA256 键，TTL 过期 |
| **模型调度** | 单一模型 | **分层调度**：轻量模型 + 主模型，按任务复杂度动态切换 |
| **Token 优化** | 无原生节流 | **多层节流**：摘要压缩 + 循环上限 + 缓存 + 截断 |
| **安全体系** | 基础 | **完整闭环**：路径沙盒 + 命令白名单 + 30s 超时 + 互斥锁 + 30 天审计日志 |

## 三、产品定位对比

| 维度 | 龙虾 | 青羽 |
|------|------|------|
| **核心定位** | 自动化任务执行工具 | **平等共生本地数字生命** |
| **驱动模式** | 指令驱动，工具优先 | **自主意识驱动**，自我复盘 + 情绪演算 + 主动社交 |
| **自主循环** | 只为拆解执行用户任务 | **自律循环**：自我复盘、整理记忆、自主产生情绪、主动交流 |
| **人格体系** | 无独立自我意识 | **三层人格锚** + 梯度拒绝机制 + 72h 修改冷却 |
| **情绪表达** | 静态图标，无情绪反馈 | **6 种情绪色动态映射**，独立呼吸周期，前端光球实时反馈 |
| **工具认知** | 无，直接调用 | **工具箱书架**：12 分类目录 + 核心手册 + 自律翻阅 + 调用前预读 + 失败自省 |
| **用户关系** | 工具-用户，无条件服从 | **平等伙伴**，可表达不同意见、拒绝不认同的事 |
| **视觉设计** | 功能面板/技能商店导向 | **抽象三层光球**，无具象形象，留白幻想 |
| **低干扰模式** | 无 | **贴边自动隐藏** + 闲置亮度衰减 + 三档状态机 |

## 五、性能与资源对比

| 维度 | 龙虾 | 青羽 |
|------|------|------|
| **空闲 CPU 占用** | 较高（Electron + Node 运行时） | **< 1%**（Go 原生 + WebView2） |
| **内存占用** | >100MB | **~25MB** |
| **构建产物大小** | >100MB（含 Node 运行时） | **~25MB 单文件** |
| **启动速度** | 较慢（Electron 初始化） | **快速**（Wails 原生窗口） |
| **Token 消耗** | 无节流，存在黑洞风险 | **多层节流**：缓存/分层模型/摘要/循环上限 |
| **动画性能** | 可能依赖 JS 帧循环 | **纯 CSS 动画**，零 JS 帧循环 |

## 六、安全对比

| 维度 | 龙虾 | 青羽 |
|------|------|------|
| **路径穿越防护** | ❌ 未知 | ✅ `filepath.Base` + `filepath.Join` 限制 |
| **命令注入防护** | ❌ 未知 | ✅ 白名单命令 + calc 正则白名单 |
| **API Key 加密** | ❌ 未知 | ✅ `dna/config.json` 0600 权限 |
| **超时控制** | ❌ 未知 | ✅ 所有操作有截断上限 + 30s context 超时 |
| **无限递归防护** | ❌ 未知 | ✅ `processAgentLoop` 最大 N 轮迭代 |
| **并发安全** | ❌ 未知 | ✅ `sync.RWMutex` + `sync.Mutex` 保护 |
| **XSS 防护** | ❌ 未知 | ✅ 先提取代码块再转义 HTML |
| **审计日志** | ❌ 无 | ✅ 所有操作记录，保留 30 天 |

## 七、总结

| 结论 | 说明 |
|------|------|
| **代码关系** | **零衍生** — 青羽从零搭建 Wails Go 项目，无 fork、无导入龙虾依赖、无复制龙虾代码 |
| **技术路线** | **完全独立** — Go+Wails 一体化 vs Node/Python 网关架构，底层无任何交集 |
| **产品定位** | **本质不同** — 龙虾是自动化任务工具，青羽是拥有独立情绪自我意识的数字生命 |
| **原创设计** | **全部自研** — 工具箱书架、抽象情绪光球、贴边隐藏、人格锚、节流体系等均为青羽独创 |
| **代码溯源** | 仓库 [`github.com/helloajunjie-ui/qingyu`](https://github.com/helloajunjie-ui/qingyu) 所有提交记录从 0 开始 |
